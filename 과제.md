## 과제 1: 현장 맞춤 Tool 3개 추가
`tools/manufacturing_tools.py`에 현장 업무 특화 Tool 3개 추가

### 요구사항
- 각 Tool에 docstring과 예시 입/출력 명시 (Args, Returns, Example 섹션 필수)
- Mock 데이터로 동작하지만 실제 API 연동 방법도 주석으로 명시

| Tool 이름 | 연동 시스템 | 설명 |

| `production_dashboard` | MES | 라인별 생산 현황 조회 |

| `training_checker` | HR | 작업자 교육 이수 여부 확인 |

## 과제 2: 복합 시나리오 Agent 실행
"라인 B 설비 이상 감지 → 부품 재고 확인 → 작업 지시서 발급" 시나리오 실행

### 요구사항
- 최소 4스텝 ReAct 루프 실행
- 각 Thought가 이전 Observation을 참조하도록 작성
- 실행 결과 스크린샷 첨부

### 예시 시나리오 구조
```python
scenario = [
    ("Thought 1: 무엇을 먼저 확인할지", "tool_name", "input"),
    ("Thought 2: 이전 결과를 보고 다음 행동 결정", "tool_name", "input"),
    ("Thought 3: ...", "tool_name", "input"),
    ("Thought 4: ...", "tool_name", "input"),
]
```

## 심화: LangSmith 연동 (API 키 있는 경우)
- `https://smith.langchain.com`에서 무료 계정 생성
- Agent 실행 trace를 LangSmith에서 확인
- 각 Thought/Action/Observation 시각화 캡처 후 제출

### LangSmith 설정
```python
import os
os.environ["LANGSMITH_API_KEY"] = "ls__your_key_here"
os.environ["LANGSMITH_PROJECT"] = "manufacturing-ai-koreatech"
os.environ["LANGCHAIN_TRACING_V2"] = "true"
```
